Overview

What is Zenith?

Zenith is an open-source algorithmic trading system built on ICT/Smart Money Concepts. It includes a Python-based strategy engine, MetaTrader 5 integration, a Flask REST API, and a real-time web dashboard.

Strategy Engine

ICT/SMC signal detection — BOS, ObZone, FVG, kill zones, confluence scoring

Backtesting

Split, walk-forward, standard modes with real MT5 OHLCV data

Web Dashboard

Real-time metrics, equity curves, trade history, live control panel

Installation

Installation

Requires Python 3.10+, MetaTrader 5 desktop client, and a broker account with MT5 access.

1 Clone the Repository

bash
git clone https://github.com/Ravenko123/TradingBot
cd TradingBot

2 Install Python Dependencies

bash
cd mt5_bot
pip install -r requirements.txt
cd ../web
pip install -r requirements.txt

3 Launch the Web API

bash
cd web
python alpha_api.py
The web server starts on http://localhost:5000 by default.
Configuration

Configuration

All bot settings live in mt5_bot/runtime_config.json. Telegram credentials are stored in mt5_bot/telegram_config.json.

1 runtime_config.json — Key Fields

  "mt5_password": "your_password",

}
Quick Start

Quick Start

Start both the trading bot and the web server to get everything running.

bash
# Terminal 1 — Start Web API
cd web && python alpha_api.py

# Terminal 2 — Start Trading Bot
cd mt5_bot && python main.py
MetaTrader 5 must be running and logged in before starting the bot.
Strategy

Signal Logic

Signals are identified using a state machine that processes M15 OHLCV candles in sequence. A trade requires a minimum confluence score of 3 from weighted sub-signals.

BOS
Break of Structure — mandatory trigger
OB
Order Block confluence (+1)
FVG
Fair Value Gap confluence (+1)
KZ
Kill Zone active (+1)
Signals with confluence < 3 are logged but never executed. Adjust min_confluence in strategy.py to tune aggressiveness.
Risk

Risk Management

Every trade is sized by ATR stop loss distance at 2% account risk per position.

Position Sizing

lots = (balance × risk%) ÷ (stop_ticks × tick_value)

Break-Even

SL moved to entry +1 pip when price reaches 1R profit

Daily Drawdown

Bot halts for the day if equity drops 5% from session open

Margin Guard

New orders blocked when margin usage exceeds 20%

Backtesting

Running Backtests

Three modes available. All use real MT5 historical data downloaded via the Python MT5 API.

1 Standard Backtest

bash
cd mt5_bot
python simple_backtest.py --symbol EURUSD --days 180

2 Improved / Walk-Forward

bash
python backtest_improved.py --mode walk_forward --symbol GBPJPY --days 180
Results are saved to mt5_bot/backtest_results/ as JSON files. The dashboard reads these automatically.

API

REST API

The Flask server exposes authenticated endpoints consumed by the dashboard. JWT optional — pass API key in the X-API-Key header.

http
Base URL: http://localhost:5000
Auth:     X-API-Key: <your_api_key>
Endpoints

Endpoints

GET/api/status

Returns bot running state, balance, equity, today's P&L, and open positions count.

GET/api/trades

Returns trade history array from SQLite. Supports ?limit=50 and ?symbol=EURUSD query params.

GET/api/performance

Aggregated performance by symbol — win rate, profit factor, total trades, net PnL.

GET/api/backtest/results

Lists all saved JSON backtest result files with summary stats.

POST/api/bot/start

Starts the trading bot subprocess. Returns {"status":"started"} or error message.

POST/api/bot/stop

Gracefully stops the bot. All open positions are closed before exit.

Help

Troubleshooting

MT5 connection fails

Ensure MetaTrader 5 is open and logged in. Verify mt5_login, mt5_password, and mt5_server in runtime_config.json match your broker credentials exactly.

No signals generated

Check that the current time falls within an active kill zone session. BOS detection requires a minimum displacement candle — low volatility periods will produce no signals.

Dashboard shows no data

Make sure alpha_api.py is running on port 5000. Check browser console for CORS errors — open directly via http://localhost:5000.

Telegram bot not responding

Confirm bot token and chat ID in telegram_config.json. Run python telegram_bot.py directly to see error output.
