Getting Started

Welcome to the Zenith Trading Bot documentation. This guide covers the ICT/SMC strategy engine, backtesting, and dashboard setup.

Note: This is an educational project. Always test on a demo account before using with real money.

What is Zenith?

Zenith is an algorithmic trading system that implements an ICT/SMC (Inner Circle Trader / Smart Money Concepts) strategy on the M15 timeframe across 7 symbols. The system features:

  • ICT/SMC state-machine signal engine with confluence scoring
  • Professional backtesting engine with real MT5 data
  • Split-sample and walk-forward validation
  • Real-time web dashboard with performance monitoring
  • Break-even trailing and ATR-based risk management
  • Telegram bot integration for alerts and remote control

Installation

Prerequisites

  • Python 3.8 or higher
  • MetaTrader 5 (for live trading)
  • MT5 account with broker support for algorithmic trading

Step 1: Clone the Repository

git clone https://github.com/yourusername/tradingbot.git
cd tradingbot

Step 2: Install Dependencies

pip install -r requirements.txt

Required Packages:

  • MetaTrader5 - MT5 Python integration
  • pandas - Data manipulation
  • numpy - Numerical computations
  • ta - Technical analysis indicators
  • python-telegram-bot - Telegram integration
  • flask - Web API backend
  • flask-cors - CORS support

Step 3: Configure MT5 Connection

Edit config/settings.py with your MT5 credentials:

SETTINGS = {
    "mt5": {
        "login": 123456789,
        "password": "your_password",
        "server": "YourBroker-Demo"
    }
}

Configuration

Strategy Configuration

The ICT/SMC strategy parameters are configured in mt5_bot/simple_backtest.py:

ICT/SMC Parameters:

EMA_FAST = 9           # Fast EMA period
EMA_SLOW = 21          # Slow EMA period
ADX_PERIOD = 14        # ADX calculation period
ADX_THRESHOLD = 25     # Minimum trend strength (Doc: <25 = consolidation)
ATR_PERIOD = 14        # ATR calculation period
ATR_MULTIPLIER = 1.0   # Stop loss = ATR x multiplier
RISK_REWARD = 2.0      # Take profit = SL x R:R
RISK_PERCENT = 2.0     # Risk 2% of account per trade
BACKTEST_DAYS = 180    # 6 months of data

Signal Confluence Scoring

Each entry is scored on 7 factors (minimum score of 3 required):

  • +1 Displacement BOS candle (strong body)
  • +1 Fair Value Gap in the impulse
  • +1 Liquidity sweep before the move
  • +1 Strong Order Block candle
  • +1 Session kill zone (London 07-10 / NY 13-16 UTC)
  • +1 Premium/Discount zone alignment
  • +1 Rejection wick on entry candle

Trading Strategy

ICT/SMC (Smart Money Concepts)

The core strategy detects institutional order flow through a multi-bar state machine:

How It Works:

  1. Trend Filter — EMA 9/21 determines bull/bear bias, ADX ≥ 25 confirms trend
  2. Break of Structure — Price breaks a confirmed swing high/low with a displacement candle
  3. Order Block Zone — Last strong opposite candle before the BOS impulse becomes an entry zone
  4. Pullback Entry — Wait for price to retrace into the OB zone with a confirmation candle
  5. Confluence Score — Score from FVG, displacement, sweep, session, premium/discount, rejection wick (≥ 3 required)
  6. Risk Management — ATR-based SL, 2R TP, break-even at 1R (SL → entry price), trailing at 50% target (SL → 25% profit). Daily drawdown circuit breaker at 5%, margin protection at 20% usage.

Supported Markets:

  • Forex: EURUSD, GBPUSD, USDJPY, GBPJPY
  • Metals: XAUUSD (Gold)
  • Crypto: BTCUSD (Bitcoin)
  • Indices: NAS100 (Nasdaq)

Backtesting

Running a Backtest

Test the strategy with historical data before risking real money:

# Split backtest (70% train / 30% test)
python mt5_bot/simple_backtest.py --mode split --split-ratio 0.7

# Walk-forward analysis (rolling windows)
python mt5_bot/simple_backtest.py --mode walk_forward

# Standard backtest (full dataset)
python mt5_bot/simple_backtest.py --mode standard

Backtest Parameters

  • --mode: Backtest mode (split, walk_forward, standard)
  • --split-ratio: Train/test split ratio (default: 0.7)
  • --symbol: Single symbol to test (optional, tests all 7 by default)

Understanding Results

The backtest engine generates comprehensive metrics:

Total Return

Overall profit/loss percentage from initial balance

Sharpe Ratio

Risk-adjusted return. Higher is better (>1.0 is good)

Max Drawdown

Largest peak-to-trough decline. Lower is better

Win Rate

Percentage of profitable trades

Profit Factor

Gross profit / Gross loss. Should be >1.5

Average Win/Loss

Average profit vs average loss per trade

Optimizing Parameters

Parameters are tuned in mt5_bot/simple_backtest.py at the top of the file. Run split backtests to validate changes:

python mt5_bot/simple_backtest.py --mode split --split-ratio 0.7

Troubleshooting

Common Issues

Bot won't connect to MT5

Symptoms: "MT5 initialization failed" error

Solutions:

  • Ensure MT5 terminal is running
  • Check credentials in config/settings.py
  • Verify MT5 allows algorithmic trading (Tools > Options > Expert Advisors)
  • Check if your broker supports API trading

No trades being placed

Symptoms: Bot runs but doesn't place orders

Solutions:

  • Check market hours (some instruments only trade certain times)
  • Verify sufficient account balance
  • Check strategy filters in strategy_config.py
  • Review logs in logs/system.log for details

Dashboard shows no data

Symptoms: Empty charts and tables

Solutions:

  • Ensure API backend is running: python web/api.py
  • Check CORS settings in api.py
  • Verify trades.csv exists and has data
  • Check browser console for JavaScript errors

Backtest results don't match live trading

Symptoms: Different performance between backtest and live

Reasons:

  • Slippage and commissions not accounted for in backtest
  • Market conditions changed
  • Different data quality (historical vs live)
  • Look-ahead bias in backtest

Solutions:

  • Add realistic slippage/commission in backtest settings
  • Test on recent data that matches current conditions
  • Use demo account first to validate results
